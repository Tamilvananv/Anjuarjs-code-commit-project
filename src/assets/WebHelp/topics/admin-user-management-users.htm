Users

 

 

 

Adding and managing users of the digification project:

The Landing page of the section will be a list of users which have been added to the digification database from Immuneel's Active directory. The logged in user may perform the following actions while on this page:

oUser may view the list, arrange in any order by clicking on the title of the column or sort and group by column by dragging and dropping the column in the panel above the table (marked green in the screen shot below).

oUser may download the user list in report format (marked blue in the below screen shot).

oUser may add a new user by clicking on 'Add User' (marked orange in the below screen shot).

oUser may edit an existing user by clicking on Edit action icon ()

oSearch a pattern by typing in the search bar

 

 

❖Adding a new user:

'Add User', on click, will take the logged-in user to the list of employees present in the active directory of the company. A new user may be added by clicking on the check box next to the user details and clicking 'Save & Next'. The new user is added on this click even if he/she isn't assigned to a department/manager or haven't been given any read-write permissions.

 

One Clicking Next, a department from the existing list of departments (refer page User Management > Departments for more details on this section) may be assigned to the user by selecting a department and clicking arrow '>' (To remove the department from the user's profile, arrow '<' may be clicked after selecting the department name if needed). The department gets linked to the user's profile on clicking 'Update and Next'.

 

 

A manager can be set for the user on the next page by checking the name of the manager from the list and clicking 'Update and Next'.

 

Additional information of Employee ID (Numeric value) and Date of Joining (From the date picker till today) may be set on the next page (This information will be useful for the Learning management system module as of now) and clicking 'update and next'.

 

 

User read/write permissions may be set on the next page. Here, a list of all features per module will be listed. A user may be assigned any of these permissions or a permission group (detailed in permission groups section). Permissions may be changed for individuals even after assigning a permission group. On clicking 'update', the said user will now be able to view/write in the corresponding sections of modules for which he/she has been granted permissions to.

 

❖Editing an existing user:

User details may be edited by clicking on the pen icon () in actions against the user name in the User management section.

 

Notes:

oUsers though present in Immuneel's active directory will start reflecting in this section only after they are manually added here (using the 'Add users' feature).

oUsers once created will not be permitted to be deleted from the system as per URS.
